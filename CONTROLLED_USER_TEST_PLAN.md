# Telegram Channel Factory — Controlled User Test Plan

## Status

Подготовлено для запуска controlled user tests на тестовом боте.

**Важно:** сам пункт backlog `63. Провести первые controlled user tests с реальными людьми на тестовом боте` считается закрытым только после реального прохождения сценария живыми участниками. В этой среде без приглашения пользователей и без внешнего контакта можно подготовить пакет, но нельзя честно подтвердить выполнение тестов.

---

## Goal

Проверить на первых реальных пользователях три вещи:
- понятен ли onboarding без помощи разработчика
- проходит ли человек путь `setup -> connect channel -> content plan -> ideas -> drafts -> publication`
- где именно возникают UX-затыки, из-за которых пользователь останавливается или просит помощь

---

## Test environment

Рекомендуемый безопасный режим:

```env
APP_ENV=staging
RUNTIME_MODE=demo
PUBLISHER_BACKEND=stub
```

Если нужен live-proof сценарий публикации, его проводить отдельным мини-тестом после основного safe-run.

## Required assets before inviting users

1. Тестовый бот доступен и отвечает на `/start`
2. Подготовлен короткий launch path для участника: `FIRST_USER_LAUNCH_PATH.md`
3. Оператор держит под рукой:
   - `FINAL_QA_RUNBOOK.md`
   - `OPERATOR_HANDBOOK.md`
   - этот файл
4. Есть шаблон фиксации результатов: `CONTROLLED_USER_TEST_RESULTS_TEMPLATE.md`

---

## Participants

Минимум для первой волны:
- 2–3 реальных пользователя
- желательно 1 человек с опытом Telegram-каналов и 1–2 человека без глубокого контекста проекта

Для каждого участника зафиксировать:
- роль / профиль
- был ли у него уже Telegram-канал
- дошёл ли он до первой публикации
- сколько раз потребовалась помощь

---

## Moderator rules

Во время теста нельзя:
- объяснять интерфейс заранее
- подсказывать следующий шаг до момента явной блокировки
- выполнять скрытые developer-only действия

Можно:
- напомнить цель теста
- попросить проговаривать мысли вслух
- фиксировать время, паузы и места, где пользователь теряется

## Test script

### Scenario A — First launch

Попросить пользователя:
1. открыть бота
2. отправить `/start`
3. создать проект
4. выбрать agent preset
5. подключить канал через `@username`
6. дойти до состояния, где понятен следующий шаг

Что фиксировать:
- понял ли пользователь, что делает бот
- заметил ли кнопку `Создать канал`
- понял ли разницу между `У меня уже есть канал` и `Как создать канал`
- справился ли с шагом подключения канала без объяснений

### Scenario B — Content pipeline

Попросить пользователя:
1. создать контент-план
2. сгенерировать идеи
3. создать черновики
4. открыть один черновик
5. подтвердить или отредактировать его

Что фиксировать:
- понимает ли пользователь следующий шаг после каждого экрана
- замечает ли CTA-кнопки
- не путается ли в терминах `идеи`, `черновики`, `публикации`

### Scenario C — Publication

Попросить пользователя:
1. создать публикацию из черновика
2. либо опубликовать сразу, либо поставить в очередь
3. позже вернуться в проект и снова открыть канал

Что фиксировать:
- понимает ли пользователь разницу между `опубликовать сейчас` и `в очередь`
- видит ли статус публикации
- может ли без помощи вернуться в канал позже

## Success criteria for closing backlog item 63

Задачу 63 можно отметить `[x]`, только если выполнено всё ниже:
- минимум 2 реальных пользователя прошли тест на тестовом боте
- по каждому есть отдельный заполненный результат
- зафиксированы конкретные UX-боли и сильные стороны
- понятен следующий backlog item по реальным наблюдениям, а не по догадкам
