# Telegram Channel Factory — Operation Modes v1

## 1. Purpose

Этот документ фиксирует режимы работы канала для MVP v1.

## 2. Supported modes

### Manual
Ничего не публикуется без явного действия пользователя.

### Semi-auto
Система может генерировать и готовить публикации автоматически, но финальное действие требует пользовательского approve или запуска.

### Auto
Система может автоматически проводить публикации по текущим правилам и расписанию.

## 3. Product rule

Режим работы должен быть видимым и легко изменяемым из Telegram UI.

## 4. UX rule

Пользователю надо объяснять режимы простыми последствиями:
- кто принимает финальное решение
- что публикуется автоматически
- что требует ручного контроля

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- MVP поддерживает 3 режима: manual / semi-auto / auto
- режим является частью конфигурации проекта
- смена режима должна быть доступна после запуска канала
