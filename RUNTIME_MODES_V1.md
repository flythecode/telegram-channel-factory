# Telegram Channel Factory — Runtime Modes v1

## 1. Purpose

Этот документ фиксирует разделение режимов среды: demo, stub, live.

## 2. Goal

Команда разработки и продукт должны чётко различать безопасный локальный режим и реальные Telegram-публикации.

## 3. Modes

### Stub
Безопасный локальный режим без реальной отправки в Telegram.

### Demo
Демонстрационный режим с контролируемым поведением, ориентированный на показ продукта.

### Live
Реальный режим с настоящими Telegram-публикациями и рабочими токенами.

## 4. Product rule

Переход между режимами должен быть явным, а не скрытым следствием случайного `.env`.

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- режимы `stub`, `demo`, `live` должны быть явно разведены
- локальная разработка не должна по умолчанию стрелять в live Telegram
- runtime mode должен быть понятной частью конфигурации системы
