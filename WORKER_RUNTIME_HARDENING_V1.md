# Telegram Channel Factory — Worker Runtime Hardening v1

## 1. Purpose

Этот документ фиксирует требования к укреплению worker/runtime слоя в MVP v1.

## 2. Goal

Worker должен устойчиво обрабатывать очередь публикаций и фоновые задачи без хрупкого поведения.

## 3. Priority areas

- предсказуемый polling
- безопасная обработка ошибок
- понятные статусные переходы
- устойчивость к временным сбоям
- корректная работа с queued/sending/sent/failed

## 4. MVP rule

Worker в MVP может быть простым, но не должен быть непрозрачным и нестабильным.

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- worker/runtime слой требует отдельного внимания перед релизом
- очередь публикаций должна быть надёжной
- фоновые процессы — часть core product behavior
