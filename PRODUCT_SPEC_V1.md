# Telegram Channel Factory — Product Spec v1

## 1. Product summary

**Telegram Channel Factory** — это продукт для создания и управления Telegram-каналами под ключ через Telegram-бота с понятным кнопочным интерфейсом.

Пользователь с помощью бота:
- создаёт проект канала
- подключает или настраивает Telegram-канал
- выбирает тему, стиль, формат и частоту контента
- назначает команду из 3–7 ИИ-агентов
- запускает генерацию контент-плана, черновиков и публикаций
- в любой момент меняет настройки, агентов, стратегию и режим работы канала

Ключевая идея продукта: это **не одноразовый setup wizard**, а **постоянная Telegram-админка для AI-управления каналом**.

---

## 2. Vision

Сделать продукт, в котором пользователь может из одного Telegram-бота:
- быстро запустить новый Telegram-канал как медиа-проект
- управлять контентной машиной без входа в код, базы или внешнюю админку
- использовать команду ИИ-агентов как редакцию канала
- менять конфигурацию канала и агентов в любой момент без пересоздания проекта

---

## 3. Who is this product for

### Primary audience
- соло-авторы и владельцы Telegram-каналов
- предприниматели, запускающие контентные каналы под бизнес/личный бренд
- small media teams
- operators / content managers, которым нужен semi-auto или auto контентный pipeline
- AI-first пользователи, которые хотят управлять каналом через Telegram, а не через сложную веб-панель

### Ideal user profile
Пользователь, который:
- хочет быстро запустить Telegram-канал под ключ
- не хочет руками собирать пайплайн из нескольких сервисов
- хочет работать через понятный бот-интерфейс
- хочет не только создать канал, но и постоянно им управлять
- хочет использовать несколько ИИ-агентов с разными ролями

---

## 4. Core problem

Сейчас запуск и ведение Telegram-канала требует вручную соединять:
- идеи и стратегию
- контент-план
- генерацию текстов
- редактуру
- публикации
- расписание
- контроль качества
- донастройку процесса по мере изменения задач

Обычно это либо хаос из нескольких инструментов, либо одноразовые AI-боты без нормального управления после запуска.

**Telegram Channel Factory** решает это, давая единое пространство управления прямо внутри Telegram.

---

## 5. Core value proposition

### Main value
Пользователь получает возможность **создать и вести Telegram-канал как систему**, а не как набор ручных действий.

### Product value
- запуск канала под ключ из Telegram
- управление через кнопки и простые сценарии
- multi-agent content pipeline
- редактируемость после запуска
- понятные режимы: manual / semi-auto / auto
- единый интерфейс для настройки, черновиков и публикаций

### Key differentiation
Главное отличие продукта:
- не просто генерация постов
- не просто bot assistant
- не просто scheduler
- а **AI control panel для Telegram-канала с командой ИИ-агентов**

## 6. Product principles

### 6.0 Tenant model

Продукт фиксирует следующую tenant-модель:
- **1 платящий клиент** может иметь **1 или больше каналов**;
- каждый канал оформляется как **отдельный `project`** (channel project);
- на каждый такой `project` приходится **своя project-scoped agent team**;
- `workspace` группирует проекты клиента, но не является shared agent runtime;
- любые generation, prompts, usage, лимиты и audit-события должны трактоваться как данные уровня `client -> project/channel`, а не как общий контекст пользователя.

Это правило нужно для того, чтобы один оператор мог вести несколько клиентских каналов, не смешивая стратегию, агентные настройки и историю исполнения между ними.

1. **Telegram-first** — весь основной пользовательский опыт должен работать внутри Telegram-бота.
2. **Setup + ongoing management** — продукт должен быть полезен и на старте, и после запуска канала.
3. **Editable by design** — пользователь должен иметь возможность менять настройки в любой момент.
4. **Multi-agent, but understandable** — сложность orchestration должна быть скрыта за простым UI.
5. **Safe automation** — автоматизация должна быть управляемой, с режимами контроля.
6. **MVP before overengineering** — в v1 нужен рабочий и понятный продукт, а не максимальная сложность.

## 7. What the product does in v1

### 7.1 User can create a channel project
Пользователь может:
- создать новый проект
- задать название проекта
- выбрать тему/нишу
- указать язык
- задать базовое позиционирование канала
- выбрать целевую аудиторию в упрощённом виде

### 7.2 User can connect a Telegram channel
Пользователь может:
- указать существующий Telegram-канал
- пройти guided flow по подключению канала
- добавить бота в админы
- завершить привязку канала к проекту

### 7.3 User can configure channel strategy
Пользователь может задать:
- tone of voice
- тип контента
- частоту публикаций
- длину/формат постов
- базовые рубрики
- режим работы канала

### 7.4 User can choose and manage AI agents
Пользователь может:
- выбрать preset команды из 3–7 агентов
- видеть роли агентов
- включать/выключать агентов
- редактировать базовые настройки агентов
- в MVP — редактировать prompts в упрощённом виде

### 7.5 User can generate and manage content
Пользователь может:
- создать контент-план
- получить список задач/идей
- сгенерировать черновики
- approve/reject черновики
- отправить в публикацию
- поставить публикации в очередь

### 7.6 User can continue managing the project after setup
Пользователь может в любой момент:
- зайти в список своих каналов
- открыть конкретный канал
- изменить настройки канала
- изменить состав агентов
- изменить частоту/режим публикации
- повторно сгенерировать контент-план или черновики
- управлять очередью публикаций

## 8. Functional scope of MVP v1

### Included in MVP v1

#### A. Telegram bot UI
- `/start`
- главное меню
- меню "Мои каналы"
- меню создания проекта
- меню управления конкретным каналом
- inline buttons
- простая навигация назад/в меню

#### B. Project and channel setup
- создание проекта
- базовая настройка канала
- подключение существующего Telegram-канала
- guided instructions по правам бота

#### C. Agent team configuration
- presets на 3 / 5 / 7 агентов
- базовые роли агентов
- просмотр состава команды
- включение/выключение агента
- базовое редактирование agent settings

#### D. Content workflow
- создание content plan
- создание tasks
- генерация drafts
- approve / reject draft
- queue publication
- публикация через backend pipeline

#### E. Operating modes
- manual
- semi-auto
- auto

#### F. Post-setup editing
- изменение основных настроек после создания
- изменение агентной команды после создания
- изменение контентных параметров после создания
- изменение расписания и режима работы после создания

#### G. Backend support
- хранение проектов, каналов, агентов, планов, задач, черновиков, публикаций
- worker для обработки очереди публикаций
- publisher abstraction
- Telegram publisher adapter

## 9. MVP boundaries

### What MVP v1 must prove
MVP v1 должен доказать 4 вещи:
1. пользователь понимает и проходит setup через Telegram-бота
2. пользователь может создать и подключить канал без боли
3. команда ИИ-агентов реально помогает собирать контентный pipeline
4. пользователь может продолжать управлять каналом после создания, а не только один раз его настроить

### What MVP v1 is not trying to prove yet
MVP v1 не обязан доказывать:
- что система закрывает все production edge cases
- что orchestration максимально интеллектуален
- что UI покрывает все редкие сценарии
- что продукт уже готов для large teams / enterprise use

---

## 11. Core user scenarios in v1

### Scenario 1 — create first channel project
Пользователь заходит в бота, создаёт проект, настраивает базовые параметры и подключает канал.

### Scenario 2 — choose agent team
Пользователь выбирает preset команды агентов под свой тип контента и включает подходящий режим работы.

### Scenario 3 — generate initial content pipeline
Пользователь получает контент-план, задачи и черновики для первых публикаций.

### Scenario 4 — approve and publish
Пользователь согласовывает черновик, ставит его в очередь и публикует.

### Scenario 5 — change settings after launch
Пользователь возвращается позже и меняет стратегию, состав агентов, частоту публикаций или режим канала.

## 12. UX requirements for MVP v1

### Must be true
- пользователь не должен теряться в навигации
- каждый экран должен вести к понятному следующему действию
- текстов должно быть мало, смысл должен быть ясен
- ключевые действия должны быть доступны через кнопки
- после setup пользователь должен легко находить путь обратно к управлению каналом

### UX style
- короткие экраны
- ясные CTA
- минимум перегруза
- понятные подписи ролей и режимов
- дружелюбные инструкции при подключении канала и прав бота

---

## 13. Success criteria for MVP v1

MVP v1 можно считать успешным, если:
- пользователь может самостоятельно пройти основной сценарий без ручного сопровождения разработчика
- можно создать проект и подключить реальный Telegram-канал
- можно выбрать и настроить команду агентов
- можно получить контент-план и черновики
- можно согласовать и опубликовать пост
- можно позже изменить настройки канала и агентов
- система остаётся понятной на уровне Telegram UI

---

## 14. Main risks

1. UI внутри Telegram станет слишком сложным и перегруженным.
2. Multi-agent logic будет мощной, но непонятной для пользователя.
3. Setup flow подключения канала окажется слишком хрупким.
4. Продукт скатится в одноразовый wizard вместо управляемой системы.
5. В MVP попадёт слишком много лишнего и релиз расползётся.

---

## 15. Product decisions fixed by this spec

Этим документом фиксируются следующие решения:
- продукт делается **Telegram-first**
- основной интерфейс продукта — **Telegram bot UI с кнопками**
- продукт должен поддерживать **3–7 ИИ-агентов**
- пользователь должен иметь возможность **редактировать настройки после создания**
- MVP v1 — это **рабочая версия управления каналом**, а не enterprise-платформа
- первый релиз фокусируется на **создании, настройке, запуске и управлении каналом**, а не на биллинге, сложной аналитике и enterprise-функциях

---

## 16. One-line product definition

**Telegram Channel Factory** — это Telegram-бот для создания и управления Telegram-каналами под ключ с командой из 3–7 ИИ-агентов, контентным пайплайном и возможностью менять стратегию и настройки после запуска.
