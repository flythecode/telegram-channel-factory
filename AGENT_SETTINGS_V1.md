# Telegram Channel Factory — Agent Settings v1

## 1. Purpose

Этот документ фиксирует, какие настройки агента должны быть доступны в MVP v1.

---

## 2. Product rule

Агент — это не статичная сущность. Пользователь должен иметь возможность менять параметры агента после создания проекта.

---

## 3. Editable agent settings in MVP

Для каждого агента в MVP должны поддерживаться:
- `display_name`
- `role_code`
- `description`
- `is_enabled`
- `sort_order`
- `system_prompt`
- `style_prompt`
- `custom_prompt`
- `model_hint` (optional)
- `temperature_hint` (optional)

---

## 4. What user should be able to do in UI

Пользователь должен уметь:
- включить или выключить агента
- поменять роль агента
- поменять порядок агента в цепочке
- изменить prompt агента
- вернуть базовые настройки preset'а

---

## 5. UX rule

В MVP не нужно перегружать пользователя десятками параметров.

Интерфейс должен показывать:
- роль агента
- краткое описание
- статус активности
- упрощённую настройку prompt'а

Расширенные параметры можно оставить под отдельную кнопку `Доп. настройки` или скрыть до post-MVP.

---

## 6. Safety rule

Изменения агентных настроек должны влиять на будущие генерации и не должны автоматически переписывать старые draft'ы.

---

## 7. Decisions fixed by this doc

Этим документом фиксируется:
- агент в MVP — редактируемый объект
- prompt'ы являются частью пользовательской конфигурации
- включение/выключение и порядок агента — обязательные функции
- изменения действуют на будущее, а не переписывают прошлые результаты
