# Telegram Channel Factory — Bot UI Tests v1

## 1. Purpose

Этот документ фиксирует требования к тестам Telegram bot UI в MVP v1.

## 2. Goal

Нужно проверять, что ключевые кнопочные пользовательские сценарии действительно работают и не ломаются при изменениях.

## 3. Priority scenarios to test

- `/start` и главное меню
- wizard создания проекта
- выбор preset команды агентов
- подключение канала
- вход в `Мои каналы`
- навигация в карточку канала
- действия с черновиками
- действия с публикациями

## 4. MVP rule

Не обязательно покрыть весь UI идеально, но ключевые входные сценарии должны быть тестируемыми.

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- bot UI flow требует отдельного тестового слоя
- критичные пользовательские сценарии должны быть проверяемыми
- Telegram UI нельзя считать корректным только по manual inspection
