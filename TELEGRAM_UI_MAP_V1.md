# Telegram Channel Factory — Telegram UI Map v1

## 1. Purpose

Этот документ фиксирует структуру Telegram UI для **Telegram Channel Factory** в рамках MVP v1.

Цель:
- определить все основные экраны бота
- зафиксировать кнопки и переходы
- описать вложенность меню
- показать, как пользователь создаёт канал и потом управляет им
- дать основу для реализации bot handlers и FSM

---

## 2. UI principles

### Core principles
1. **Telegram-first** — пользователь решает основные задачи внутри Telegram.
2. **Buttons-first** — ключевые действия доступны через inline buttons.
3. **Short screens** — минимум перегруженного текста.
4. **Clear next step** — на каждом экране должен быть понятен следующий шаг.
5. **Editable flows** — пользователь может не только создать, но и потом менять настройки.
6. **Safe navigation** — всегда есть путь назад, в карточку канала или в главное меню.

---

UI делится на 3 уровня:

### Level 1 — Global
- Start / onboarding

- Мои каналы

### Level 2 — Channel workspace
- Обзор канала
- Настройки канала
- Агенты
- Контент-план
- Черновики
- Публикации / очередь

### Level 3 — Item detail
- Карточка агента
- Карточка черновика
- Карточка публикации
- Карточка контент-плана / задачи
- Экран изменения параметра

## 18. Navigation rules

### Required navigation buttons
Почти на всех экранах должны быть:
- `Назад`
- `Главное меню`
или
- `Назад к каналу`

### Navigation rules
1. Из глубокой карточки пользователь всегда может вернуться на уровень выше.
2. Из channel-level экранов пользователь всегда может вернуться в карточку канала.
3. Из setup flow пользователь может отменить сценарий.
4. После ключевых действий бот должен показывать следующий очевидный CTA.
5. После создания проекта пользователь должен попадать не в тупик, а в следующий логичный шаг.

## 19. Callback/data naming guidance

Для будущей реализации callback_data стоит придерживаться понятной схемы:
- `main:create_channel`
- `main:my_channels`
- `channel:open:{id}`
- `channel:settings:{id}`
- `agent:list:{channel_id}`
- `agent:open:{agent_id}`
- `draft:open:{draft_id}`
- `draft:approve:{draft_id}`
- `pub:open:{publication_id}`
- `mode:set:{channel_id}:manual`

Это не финальная спецификация callback_data, а ориентир на уровне UI map.

## 21. Completion criteria

Telegram UI Map v1 считается достаточной для старта реализации, если по этому документу уже можно:
- собрать меню и структуру экранов

- определить API-запросы, которые понадобятся с каждого экрана
- не спорить каждый раз заново, как пользователь будет ходить по продукту
