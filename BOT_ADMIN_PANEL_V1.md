# Telegram Channel Factory — Bot Admin Panel v1

## 1. Purpose

Этот документ фиксирует принцип постоянной админки внутри Telegram-бота.

---

## 2. Core rule

После завершения setup пользователь должен получить не одноразовый результат, а постоянный интерфейс управления каналом.

---

## 3. Core sections of admin panel

У каждой карточки канала в MVP должны быть разделы:
- Обзор
- Настройки
- Агенты
- Контент-план
- Черновики
- Публикации
- Режим работы

---

## 4. Admin panel responsibilities

Через админку пользователь должен уметь:
- менять стратегию канала
- менять настройки агентов
- генерировать новый контент
- управлять черновиками
- управлять очередью публикаций
- видеть статус тарифа, остаток generation-лимита и причину блокировки при превышении квоты
- ставить канал на паузу и запускать снова

---

## 5. MVP rule

Админка должна ощущаться как основной рабочий режим продукта, а wizard — только как входной шаг.

---

## 6. Decisions fixed by this doc

Этим документом фиксируется:
- Telegram bot admin panel — обязательная часть MVP
- setup flow не является основным режимом продукта
- управление после создания канала — ключевая часть продукта
