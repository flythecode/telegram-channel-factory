# Telegram Channel Factory — Staging / Demo Scenario v1

## 1. Purpose

Этот документ фиксирует требования к стабильному staging/demo сценарию продукта.

## 2. Goal

Нужен предсказуемый способ показать продукт без скрытых ручных костылей и импровизации.

## 3. Demo should prove

- понятный `/start`
- рабочий setup flow
- выбор агентной команды
- подключение канала
- генерацию контента
- управление после создания

## 4. MVP rule

Demo/staging сценарий должен быть воспроизводимым и безопасным.

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- demo path требует отдельной подготовки
- staging сценарий нужен для проверки и показа продукта
- демонстрация должна опираться на стабильную конфигурацию
