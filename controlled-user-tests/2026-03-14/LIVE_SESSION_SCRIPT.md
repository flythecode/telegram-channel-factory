# Live session script — first controlled user test wave

## 0. Opening (30–60 sec)

Скажи участнику:

> Спасибо, что помогаешь. Мы тестируем не тебя, а интерфейс. Постарайся идти без подсказок и говорить вслух, что ты думаешь. Я буду вмешиваться только если ты реально застрял.

## 1. Warm-up question

- Был ли у тебя раньше опыт с Telegram-каналами?
- Создавал ли ты когда-нибудь публикации или контент-планы?

## 2. Scenario A — first launch

Prompt:

> Открой бота, нажми /start и попробуй начать работу так, как если бы ты делал это впервые.

Moderator records:
- заметил ли пользователь главный CTA
- как интерпретирует первые кнопки
- где впервые замедлился

## 3. Scenario B — content pipeline

Prompt:

> Теперь попробуй дойти до черновика публикации: создай контент-план, идеи и черновики, потом открой один черновик.

Moderator records:
- понимает ли разницу между сущностями
- видит ли следующий шаг после каждого экрана
- просит ли объяснить терминологию

## 4. Scenario C — publication and return later

Prompt:

> Создай публикацию из черновика, выбери — опубликовать сейчас или поставить в очередь, а потом попробуй вернуться к каналу ещё раз так, будто зашёл позже.

Moderator records:
- понимает ли пользователь разницу between publish-now and queue
- может ли без помощи найти канал снова
- замечает ли статусы

## 5. If participant is blocked

Allowed moderator fallback:
- сначала спросить: «Что ты ожидал увидеть здесь?»
- затем: «Какая следующая кнопка кажется тебе наиболее вероятной?»
- только после этого дать минимальную подсказку

## 6. Closing questions

- Что было самым непонятным местом?
- Где интерфейс был наиболее понятным?
- Что ты ожидал увидеть, но не увидел?
- Смог бы ты в следующий раз пройти это без помощи?

## 7. Immediate wrap-up

Сразу после созвона / сессии:
- заполни participant file
- проставь severity по observation matrix
- выпиши 1 главный blocker и 1 strongest positive signal
