# Telegram Channel Factory — Final QA v1

## 1. Purpose

Этот документ фиксирует финальный ручной и полуручной QA-прогон перед релизом.

## 2. Goal

Перед выпуском продукт должен быть проверен глазами реального пользователя, а не только через код и отдельные тесты.

## 3. Core QA path

- пройти `/start`
- создать проект
- выбрать команду агентов
- подключить канал
- сгенерировать контент-план
- сгенерировать черновик
- approve draft
- опубликовать / запланировать пост
- вернуться позже и изменить настройки

## 4. MVP rule

Ручной QA обязателен даже при наличии автоматических тестов.

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- финальный пользовательский прогон обязателен
- QA должен проверять связность UX, а не только техническую корректность
- управление после setup обязательно проверяется отдельно
