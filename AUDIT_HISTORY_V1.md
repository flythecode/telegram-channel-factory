# Telegram Channel Factory — Audit & History v1

## 1. Purpose

Этот документ фиксирует базовую модель истории изменений для MVP v1.

## 2. Goal

Система должна уметь сохранять, кто и когда менял ключевые настройки проекта, команды агентов и режима работы.

## 3. What to record in MVP

Минимально стоит логировать:
- изменение стратегии проекта
- изменение operation mode
- изменение состава агентов
- изменение prompts
- изменение расписания
- ключевые действия с публикациями

## 4. MVP rule

История в MVP может быть простой, но не должна отсутствовать полностью.

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- audit/history нужен уже в MVP хотя бы в базовом виде
- изменения конфигурации должны оставлять след
- история важна для доверия и управляемости продукта
