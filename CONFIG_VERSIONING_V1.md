# Telegram Channel Factory — Config Versioning v1

## 1. Purpose

Этот документ фиксирует базовый подход к версионированию конфигурации проекта.

## 2. Goal

Когда пользователь меняет стратегию, агентов или режим канала, система должна понимать, какая конфигурация является текущей и какие изменения были сделаны.

## 3. MVP approach

- у проекта есть `current_config_version`
- при значимых изменениях создаётся новый snapshot
- новые задачи используют текущую версию конфигурации
- старые объекты не переписываются автоматически

## 4. Product rule

Config versioning нужен не ради сложности, а ради понятности и устойчивости editable продукта.

## 5. Decisions fixed by this doc

Этим документом фиксируется:
- editable продукт требует версионирования конфигурации
- snapshot-based подход подходит для MVP
- изменения должны быть направлены в будущее, а не ломать прошлые результаты
